# Tenders API Contract

## Base URL
- Use `VITE_API_BASE_URL` from environment
- Default in frontend: `http://localhost:3000/api/v1`

## Tender Schema

### Tender (response object)
```json
{
  "id": 101,
  "title": "Supply of IT Equipment and Software Licenses",
  "description": "Procurement of computers, servers, and enterprise software licenses for government offices.",
  "closingDate": "2026-07-15",
  "documentUrl": "/documents/tender-001.pdf"
}
```

### Create/Update Request Payload
```json
{
  "title": "Supply of IT Equipment and Software Licenses",
  "description": "Procurement of computers, servers, and enterprise software licenses for government offices.",
  "closingDate": "2026-07-15",
  "documentUrl": "/documents/tender-001.pdf"
}
```

## Endpoints

### 1) List Tenders
- Method: `GET`
- Path: `/tenders`
- Success: `200`
- Response body options supported by frontend:
  - Array: `[Tender, ...]`
  - Wrapped: `{ "data": [Tender, ...] }`

### 2) Create Tender
- Method: `POST`
- Path: `/tenders`
- Body: Create request payload
- Success: `201` or `200`
- Response body options supported:
  - `Tender`
  - `{ "data": Tender }`
- Notes:
  - `id` must be generated by backend

### 3) Update Tender
- Method: `PUT`
- Path: `/tenders/:id`
- Body: Update request payload
- Success: `200`
- Response body options supported:
  - `Tender`
  - `{ "data": Tender }`

### 4) Delete Tender
- Method: `DELETE`
- Path: `/tenders/:id`
- Success: `200` or `204`
- Response body optional

## Validation Rules
- `title`: required, non-empty string
- `description`: required, non-empty string
- `closingDate`: required, `YYYY-MM-DD`
- `documentUrl`: required, non-empty string (path or absolute URL)

## Frontend Behavior
- Optimistic create/update/delete is enabled in Admin Tender Management
- On API failure, UI rolls back to previous state and shows an error message
- On success, backend response is used to sync final `id`
- Tender list page auto-updates through shared local state + storage event

## Auto Archive Rule (frontend display logic)
- Tender remains active through closing date + 1 day
- From next day onward, it is shown in archived list
